# NovaTerm Sustainability and Growth Plan
**Date:** March 2026 | **Author:** NovaTerm Research

---

## License: GPL v3

Entire project under GPL v3 (compatible with Termux core).
- Doesn't prevent Play Store sales
- Doesn't prevent freemium model
- Simplest legal approach (one license for everything)
- Phase 2 Rust code can potentially use different license (new code, no Termux)

## Financial Model (3 Phases)

### Phase 0 (Now — Pre-launch): $0
- Focus on product, not money
- Set up GitHub Sponsors (zero effort)

### Phase 1 (Launch — 1K users): Donations
- GitHub Sponsors: $5, $15, $50/month tiers
- Ko-fi for one-time donations
- Open Collective when community grows
- Target: $200-500/month

### Phase 2 (1K-10K users): Freemium
- Core: free and open source everywhere
- "NovaTerm Pro" IAP on Play Store: $4.99-9.99 one-time
  - Premium themes
  - Config sync between devices
  - Advanced widgets
  - Unlimited session profiles
  - Early access to new features
- NEVER put essential features behind paywall

### Phase 3 (10K+ users): Corporate Sponsors
- Sponsor badges in README
- $500-2000/month from companies using NovaTerm

## Governance: BDFL

- One maintainer, clear decision making
- "This is my project, contributions welcome"
- Evolve only when regular contributors appear
- Strict CONTRIBUTING.md: "Open issue BEFORE starting PR"

## Avoiding Burnout

### Rules (learned from WezTerm's decline)
1. Don't respond to issues same day (unless crash)
2. "Not planned" is valid. Close without guilt
3. Set schedule: "Saturdays + 2 weeknights"
4. Automate everything (CI, releases, issue templates)
5. Don't accept contributors until it hurts
6. First contributor = someone who fixes bugs, not adds features
7. At $500/month sponsors → hire punctual help (reviews, docs)
